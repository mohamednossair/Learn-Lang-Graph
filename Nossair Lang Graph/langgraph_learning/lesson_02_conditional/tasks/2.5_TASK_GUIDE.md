### 4. Implement Processing with Retry (Retry Pattern)

**Goal:** Process document with simulated flaky behavior, retry on failure.

**State fields:**
- `processed: bool` - True if processing succeeded
- `attempt: int` - Current retry attempt (starts at 0)
- `max_attempts: int` - Maximum retry attempts (default: 3)
- `error: str` - Error message if processing failed

**Implementation:**
```python
def process_document(state: DocumentState) -> dict:
    # Simulate 70% success rate
    success = random.random() > 0.3
    if success:
        return {"processed": True, "error": ""}
    else:
        return {"processed": False, "error": "Processing failed"}
```

**Routing after processing:**
```python
def route_after_processing(state) -> Literal["react_agent", "increment_attempt", "retry_exhausted_handler"]:
    if state.get("processed", False):
        return "react_agent"  # Continue to extraction
    if state.get("attempt", 0) < state.get("max_attempts", 3):
        return "increment_attempt"  # Retry
    return "retry_exhausted_handler"  # Give up
```

**Increment attempt node:**
```python
def increment_attempt(state: DocumentState) -> dict:
    return {"attempt": state.get("attempt", 0) + 1}
```

### 5. Implement ReAct Loop (Tool-Calling Pattern)

**Goal:** Simulate tool-calling agent for data extraction.

**State fields:**
- `tool_calls_made: int` - Number of tool calls (starts at 0)
- `extracted_data: dict` - Data extracted by tools

**Implementation:**
```python
def react_agent(state: DocumentState) -> dict:
    tool_calls = state.get("tool_calls_made", 0)
    extracted = state.get("extracted_data", {}).copy()
    
    if tool_calls == 0:
        extracted["basic_info"] = f"Extracted from {state['doc_type']}"
        return {"tool_calls_made": 1, "extracted_data": extracted}
    elif tool_calls == 1:
        extracted["entities"] = ["Company A", "Company B"]
        return {"tool_calls_made": 2, "extracted_data": extracted}
    else:
        return {"tool_calls_made": tool_calls}  # No change
```

**Routing in ReAct loop:**
```python
def route_react_loop(state) -> Literal["react_agent", "route_to_final"]:
    if state.get("tool_calls_made", 0) < 2:
        return "react_agent"  # Continue loop
    return "route_to_final"  # Exit loop
```

### 7. Build the Graph

**Graph structure:**
```

                                        process_document ← validation_error_handler

                                          ↓
                                    route_to_final → handlers → END
```

**Implementation steps:**
1. Add all nodes to graph builder
2. Add START → classify_document_type edge
3. Add sequential edges: classify_document_type → classify_subtype → validate_document
4. Add conditional edge after validate_document (validation gate)
5. Add conditional edge after process_document (retry logic)
6. Add edge: increment_attempt → process_document (creates retry loop)
7. Add conditional edge after react_agent (ReAct loop)

## Common Pitfalls & Solutions

| Pitfall | Symptom | Solution |
|---------|---------|----------|
| Missing default in routing | `InvalidUpdateError` | Always add final `return "default"` |
| Using `state["key"]` | `KeyError` on missing key | Use `state.get("key", default)` |
| No recursion limit | Infinite loop | Set `config={"recursion_limit": 20}` |
| Wrong node name in mapping | `Node not found` error | Check spelling matches registered names |
| Lambda routing | Untestable, hard to debug | Extract to named function |
| LLM call in routing | Slow, unpredictable | Do LLM work in node, routing reads result |
